# OpenCode Compaction — пояснення

## Що таке компакція

Коли ти довго працюєш в одній сесії, контекст (пам'ять моделі) заповнюється. OpenCode запускає **компакцію** — стара частина діалогу стискається в короткий переказ (самарі), щоб звільнити місце для нової роботи.

Проблема: після компакції модель **забуває деталі** — точні імена файлів, номери рядків, помилки, домовленості, які були раніше.

---

## Налаштування в `opencode.jsonc`

### `compaction.auto: true`

Автоматична компакція. Коли контекст майже заповнений — OpenCode сам запускає стиснення.

- `true` (за замовчуванням) — ввімкнено
- `false` — компакція тільки по `/compact` вручну

---

### `compaction.tail_turns: 10`

**Найважливіше.** Скільки останніх повідомлень залишати **без змін** (дослівно), а не стискати в самарі.

- `2` (за замовчуванням) — лише 2 останні повідомлення verbatim
- `10` — 10 останніх повідомлень verbatim

Чим більше — тим більше контексту використовується, але тим краще модель пам'ятає що ви щойно обговорювали.

---

### `compaction.prune: false`

Prune — це коли OpenCode замінює старі результати виконаних команд (grep, read, edit, bash) на заглушку `[Old tool result content cleared]`.

- `true` — вмикає
- `false` (рекомендовано) — вимикає

**Чому false:** У OpenCode хардкоджений ліміт `PRUNE_PROTECT = 40_000` токенів. Для 200K контексту це означає що він почне різати тул-результати вже коли заповнено ~20% контексту. Тобто модель починає "забувати" що вона робила набагато раніше ніж треба.

---

### `compaction.reserved: 20000`

Резерв токенів. OpenCode запускає компакцію коли поточний контекст досягає `ліміт_моделі - reserved`.

Формула: `200000 - 20000 = 180000`

Тобто компакція почнеться на ~180K токенах, а не на 195K. Це дає запас для самого процесу компакції (який теж споживає токени), щоб не вилетіти за ліміт.

- Якщо поставити `0` — ризик що компакція сама вилетить за ліміт
- Якщо поставити `50000` — компакція почнеться раніше, на ~150K. Корисно якщо модель починає тупити раніше ліміту.

---

### `compaction.preserve_recent_tokens: 8000`

Максимум токенів які можна витратити на збереження останніх повідомлень (tail_turns).

- Якщо останні 10 повідомлень займають < 8000 токенів — всі зберігаються verbatim
- Якщо > 8000 — OpenCode обрізає найстаріші з них

**Але є баг #31769:** в конфігах де є тільки `$schema` без старих V1-полів, це значення ігнорується. Де-факто зараз працює дефолт який обмежує бюджет між 2000 і 8000 в залежності від моделі. Фікс ще не замержили.

## Плагіни

### `plugin: ["opencode-session-recall"]`

Це npm-пакет який додає агенту дві можливості:

1. **Тули для пошуку по історії** — після компакції модель може шукати інформацію в старих сесіях (як Claude Code). Працює через SQLite, не потребує окремої БД.

2. **compactionRecall** — перед компакцією плагін витягує ключові знахідки з поточної сесії і додає їх в самарі, щоб вони не загубились.

Це частково компенсує те що OpenCode не вміє робити після компакції (на відміну від Claude Code):
- Перечитувати файли назад
- Переоголошувати інструменти
- Мати pre-compact хук для збереження стану

---

## Підсумок: як це наближає до Claude Code

| Що робить Claude Code | Як це вирішено в OpenCode |
|---|---|
| Тримає останні повідомлення verbatim | `tail_turns: 10` |
| Перечитує файли після компакції | **нема** (але `opencode-agents-sync` може оновлювати AGENTS.md) |
| Має pre-compact хук | **нема** (ще не замержили) |
| Може шукати по історії | `opencode-session-recall` або `opencode-history-search` |
| Саммари втрачає менше деталей | `experimental.session.compacting` хук для кастомного промпту |

## Як відкотити зміни

### Відкотити конфіг

Повернути оригінальний конфіг (без компакції і плагінів):

```json
{
  "$schema": "https://opencode.ai/config.json"
}
```

**Файл:** `~/.config/opencode/opencode.jsonc`

### Видалити DCP-плагін

```bash
npm uninstall -g @tarquinen/opencode-dcp
```

Він також сам додав себе в `tui.json` — можна прибрати руками.

### Видалити opencode-session-recall та opencode-working-memory

Просто прибрати їх з `plugin` масиву в `opencode.jsonc`. Вони встановлюються OpenCode автоматично при наступному запуску.

### Видалити файли пам'яті

```bash
rm -rf ~/.local/share/opencode-working-memory
rm -rf ~/.local/share/opencode/session-recall
```

### Видалити цей файл

```bash
rm opencode-compaction-guide.md
```
